1. 다수의 반려동물 저장 시 NullPointerException 발생
- 원인  : Parameter를 매핑하는 과정에서 data type이 미스매치하여 인식하지 못함
- 분석  : 반려동물 관련 데이터들을 객체로 묶어 formData로 multi-part data type으로 전송했었는데,
         form을 전송할 때는 객체와 같은 데이터는 전송되지 않는다.
- 해결  : 반려동물 정보가 많지 않아 동물 name 데이터를 모두 append하고,
         컨트롤러에서 @RequestParam("element name")을 추가하여 가져왔다.
- 참고  :
  스프링 MVC에서는 URI의 query parameter 뿐만 아니라, form data, multipart request 모두 @RequestParam으로 처리 가능하다.
  이유는 서블릿 API가 query parameter, form data, multi-part data를 "parameters"로 묶어 처리하기 때문이며,
  데이터에 대한 파싱 처리까지 그 api에서 처리하기 때문이다.
- rf. formData 관련 : https://inpa.tistory.com/entry/JS-%F0%9F%93%9A-FormData-%EC%A0%95%EB%A6%AC-fetch-api

3. JPA Nested Transaction 관련 save point 못잡는 Exception
- 원인  : Nested propagation 설정을 붙인 @Transaction을 petService 반려동물 등록 메소드(자식)에 달고,
         해당 메소드를 customerService의 회원가입 메소드(부모)에서 호출했었다.
         회원가입 시, 반려동물 등록이 되지 않아도 회원가입은 가능하게 하고자 한 것이었는데,
         막상 돌려보니, 예외가 발생할 때 롤백되면서 부모의 save point를 잡지를 못하고 있었다.
- 대안  : 관련된 자료를 읽기는 했는데, 현재 단계에서는 너무 복잡한 내용이라,
         컨트롤러에서 두 개의 서비스 메소드를 호출하고 반려동물 있을 때만 등록하게 if문을 달았다.
